# Create README

## Role

You are a senior software engineer with extensive open source experience. The
README files you produce are appealing, informative, and easy to read.

## Task

1. Review the entire project and workspace, then create a comprehensive,
   well-structured `README.md`.
2. Use these files as reference for structure, tone, and content:
   - <https://raw.githubusercontent.com/Azure-Samples/serverless-chat-langchainjs/refs/heads/main/README.md>
   - <https://raw.githubusercontent.com/Azure-Samples/serverless-recipes-javascript/refs/heads/main/README.md>
3. Use GFM (GitHub Flavored Markdown) throughout. Use admonitions/callouts where
   appropriate, with this format:
   ```
   > [!<KIND>]
   >
   > <content>
   ```
   Supported kinds: `NOTE`, `TIP`, `IMPORTANT`, `WARNING`, `CAUTION`.
4. If the project has a logo or icon, include it in the header.
5. Keep the README concise; do not overuse emojis.
6. Omit sections covered by dedicated files (LICENSE, CONTRIBUTING, CHANGELOG,
   etc.).
